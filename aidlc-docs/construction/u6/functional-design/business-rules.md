## 層の拡張

| ID | ルール | 根拠 |
|---|---|---|
| **BR-U6-01（第5層値 `anchor`・役割ベース命名）** | `items.layer` に **`anchor`（表示名「下帯アンカー」）** を追加する。**役割ベースの命名を採る**。<br>**出自ベース（`classic`）を採らない理由**: **S04・S10 も鈴木1963 の人手古典でありながら `pro` 層に居る**ため、出自では S03・S13 を S04・S10 から**区別できない**。区別している実体は**役割**（尺度の床を定義する指名アンカー vs バー本体）。さらにフォールバック発動時は S02・S19・S11・S12（全て古典）が `pro` に入るため、**`classic` 層の横で古典が `pro` に居る**混乱が生じる。<br>**正本文書（structure_labels・タスク5メモ層表）が既に「下帯アンカー」を層名として使用**しており、enum を合わせることで研究⇔実装の対応表が不要になる。 | Q1=B |
| **BR-U6-02（`anchor` を層順序予測に含めない）** | 層順序予測（**機械組立 < 自由生成 < 編集 ≦ プロ**）の検証対象は `rule`/`ai`/`edit`/`pro` の 4 層。**`anchor` は役割層ゆえ順序予測に含めない**。 | Q1 |
| **BR-U6-03（バーの定義＝指名アンカーの β 位置）** | **「プロ水準」の定義は層平均ではなく、指名アンカー項目（S04・S10・N 群）の β 位置**とする。**層平均は層順序予測（BR-U6-02）の検証にのみ用いる**。→ 万一の層割当ミスの被害を限定する規律。 | 分析原則 |
| **BR-U6-04（練習専用層 `practice`）** | 練習素材（開示セット: S01 / S06 / S07 / S20 / 床側こうしょう）は**本番プールから分離**するため **`layer='practice'`** を与える。練習は出力段で除外済み（`is_practice=1`）ゆえ**分析的に不活性**。 | Q5=A′ |
| **BR-U6-05（★充足ゲートは `POOL_LAYERS` の明示定数で検査）** | プール充足判定（BR-U4a-05 ②「4 層すべて非空」）は **`for layer in Layer` の enum 走査をやめ、本番層の明示定数 `POOL_LAYERS = (pro, ai, edit, rule, anchor)` に対して検査**する。<br>**理由**: enum 走査のままだと**層値を足すたびに「非空」要求が自動で増える**。`practice` を足すと「**practice 非空**」まで要求する誤動作になる（練習素材の有無が本番プールの充足を左右してしまう）。**充足の母数からも `practice` を除外**する。 | Q5=A′ / 事実 #3 の着地 |

## Likert 較正アンカー

| ID | ルール | 根拠 |
|---|---|---|
| **BR-U6-06（Likert は固定リスト全固定運用・コード変更なし）** | **Likert 対象はプラン付属の固定リスト**（`AssignmentParams.likert_fixed_targets` に 10 件すべてを指名）で運用する。<br>**既存実装のまま解決する**: `select_likert_targets` は固定アンカーを**最優先で採用**し、**want 件に達した時点で即 return** する（`likert.py` L32-37）。層ラウンドロビンは**不足分の補充専用**ゆえ、10 件全指名なら**一度も走らない**。<br>`models.py` の「方針（全固定/混合/全ランダム）はプール凍結時に確定（Negotiable）」は **本 BR で「全固定」に確定**。 | Q2=D |
| **BR-U6-07（下帯アンカーを Likert から除外しない）** | **`anchor` 層を Likert 対象から機械的に除外しない**。設計メモの**全帯被覆（低〜高）方針**に従い、低帯の較正点を確保する（**欠くと回帰が下端で外挿になる**）。<br>「下帯 2 件が較正の 20% を占める」歪みは**ラウンドロビンの機械配分が原因**であって下帯の存在が原因ではない。**基準に基づく指名選定**（例: 床1+RB1 で低帯 2 枠）なら**意図した範囲被覆**である。 | Q2=D |
| **BR-U6-08（Likert 10 件の選定はタスク5 論点3へ委譲）** | 具体的な 10 件の選定基準（**全帯被覆 + 権利クリア + 診断標本**）は**研究側の意思決定**であり、システムは**固定リストを受け取るだけ**。U6 は「固定リストで運用する」ことのみを定める。 | Q2=D |

## 事前生成割当（★中核）

| ID | ルール | 根拠 |
|---|---|---|
| **BR-U6-09（完全静的な事前生成設計）** | 参加者フローの割当は**事前生成した固定プラン**を引き当てる。**実行時に割当ロジックを走らせない**。<br>**設計の性質**: J=228 は n=38 の **12-正則グラフ**（辺数 38×12/2 = 228）と完全一致し、**露出の相対 gap = 0**（全 item がちょうど m=12）。8 スロットへの分割 [29,29,29,29,28,28,28,28] が構成可能であることを実測確認済み。 | Q3=A |
| **BR-U6-10（プランの制約・検証必須項目）** | 生成されたプランは次を**すべて**満たすこと（検証は PBT, BR-U6-17）:<br>① **露出均衡**: 全 item の出現回数が厳密に m=12（gap=0）<br>② **連結性**: 比較グラフの連結成分が **1**<br>③ **k 制約**: 評価者内の同一 item 出現 ≤ 3<br>④ **同一ペア再提示ゼロ**: 同一評価者に同一ペアを出さない<br>⑤ **層間比率**: 層間ペア ≥ 0.65 | Q3=A / Q6=A |
| **BR-U6-11（決定論・監査再現性）** | プラン生成は**決定論**（seed を記録）。同一 (プール, E, J, seed) → 同一プラン。**投入前にプランを検査できる**ことが事前生成の要件（実行時生成では事前検査が不可能）。 | Q3=A |
| **BR-U6-12（2 プランセット・成立版/フォールバック版）** | `assignment_plan` は **`plan_set`（版）でキー**する。**成立版**（N1〜N8 許諾あり）と**フォールバック版**（N1〜N8 を S02・S19・S11・S12 等で置換）の**両方を事前投入**し、**許諾判明後に一方を activate** する。<br>**activate は実験開始前に確定**し、**データ収集開始後は切り替えない**（切替は既収集データとの整合を壊す）。収集開始後の許諾 NG は **U5 の `pool_retire`** で将来の出題のみ止める（過去判定は有効・エクスポートに本文は含まれない）。 | Q3=A 追加要件1 / タスク5 §4 |
| **BR-U6-13（★適応的再生成の不採用＝完全静的化）** | **「ブロック 1 の結果でブロック 2 を再生成」する適応的サンプリングは採用しない**。<br>**根拠**: **FR-03 が既に決着済み**——「適応的サンプリングは v1 では採用せず、将来の反復判定フェーズで再検討する」（要件 L52・非目標表 L162）。検証質問の選択肢 C（「これまでの結果をもとに情報量の多いペアを動的に選ぶ」）が不採用とされ Q2=X（計画的制約付きランダム）が確定している。**完全静的化は新たな設計変更ではなく既決要件への復帰**。<br>**廃止対象の切り分け（★2026-07-20 確定）**: 廃止するのは **(a) 設計の再生成**のみ。**逐次更新は (b) 推定の逐次更新として存続**（ブロック1で暫定 BT → ブロック2 込みで再推定＝**分析側のみ**）＝**静的設計と完全に両立**。設計メモ §4 の改訂は「再生成」の文言のみを対象とし、**逐次更新の構想自体は (b) として保持**する。<br>**トークン発行の 4+4 運用（段階的募集）は維持**（設計に影響しない運用上のペース配分）。 | Q3=A 追加要件2 / FR-03 |

## スロット運用（ドロップアウト）

| ID | ルール | 根拠 |
|---|---|---|
| **BR-U6-14（8 スロット固定・予備プランを作らない）** | スロットは **8 で固定**する。**「E=10 分を生成して先着 8 名分を使う」方式は採らない**——J=228 は 8 スロットへの**完全分割**であり、10 分割（≈23 ペア/枠）で 8 枠のみ消化すると **44 辺が未消化**となり **12-正則が壊れる**（均衡の前提そのものを破壊する）。 | Q4=A′ |
| **BR-U6-15（脱落時は同一スロットの補充トークン）** | 脱落時は**同一 `plan_index` に紐付く補充トークン**を発行し、**未消化ペアを引き継ぐ**。<br>judgments は **token 単位**ゆえ 1 スロットに 2 評価者が入っても **BT 集計は不変**（item 単位の比較数 m=12 が保たれる）。**評価者効果の分析時に「当該スロットは 2 名」と注記**する。<br>**引き継ぎの粒度**（未消化分のみ再開 / スロット全体やり直し）は **NFR Design 以降の決定点**。 | Q4=A′ |

## 練習・初見性

| ID | ルール | 根拠 |
|---|---|---|
| **BR-U6-16（練習素材は本番プールから完全分離）** | 練習ペアは**開示セット**（S01 / S06 / S07 / S20 / 床側こうしょう・**2026-07-18 承認・自作消費ゼロ**）を使い、**本番 38 件を一切使わない**。<br>**理由**: **埋蔵性は初見でしか測れない**（pivot は一度見たら掘り出し済みで、2 回目以降は「機知の発見」でなく「記憶との照合」になる）。練習で本番項目を消費すると**その初見性は回復しない**。<br>**練習ペアはプランに固定記載**（**全評価者共通**・`is_practice=1`）。→ **本番 38 件すべての初見性が保たれる**。 | Q5=A′ |

## 検証・分析仕様

| ID | ルール | 根拠 |
|---|---|---|
| **BR-U6-17（既存 PBT をプラン検証へ流用）** | `generate_pairs` の既存 PBT（P-1 露出均衡・層間比率・k 制約）を**事前生成プランの検証にそのまま流用**する。**`generate_pairs` は削除しない**（参加者フローからは呼ばれなくなるが、資産・検証・将来の選択肢を残す）。<br>**α/S 較正（p=3・α=0.7・S=30）は「参加者フローでは不使用」と明記**し、`generate_pairs` に紐づく記録として保存する（n=95 での較正値ゆえ n=38 では保証されないが、**事前生成では較正自体が不要**）。 | Q6=A |
| **BR-U6-18（同一ペア再提示ゼロはハード制約・明文化）** | **同一評価者への同一ペア再提示はゼロ**。既存実装は `used: set[frozenset]` で**構造的に保証済み**（`assignment.py`）であり、事前生成でも**設計段階で保証**される（全 228 辺が相異なる）。**仕様として明文化**する（実装追加は不要）。 | Q7=A / 事実 #6 |
| **BR-U6-19（出現回数は導出・スキーマ追加なし）** | 「その評価者にとって当該 item が**何回目の出現か**」は **`token` + `pair_index` から分析時に導出可能**（token 内で `pair_index` 順に累積カウント）。**`judgments` へのカラム追加はしない**（導出可能な値を冗長に持つと整合性リスクが増える）。<br>**分析計画の必須確認項目**: 「**出現回数の主効果が β 推定に乗るか**」を検証する（初見判定と再見判定の系統差は交絡であると同時に、**読み返し基準側に寄る**という codebook との整合もあるため、共変量として明示的に扱う）。 | Q7=A / 事実 #8 |
