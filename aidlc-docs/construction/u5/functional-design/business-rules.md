# U5 Business Rules — 出題停止（item retirement）

**背景**: **著作権上の配慮**により投入済み作品の一部を今後出題しない必要が発生。確定要件 = **物理削除は不要 / それまでの判定結果は有効のまま / 進行中セッションへの反映は不要（新規セッションのみ）**。
U1〜U4b を前提に、U5 固有の規則を **BR-U5-NN** で番号付けする。FD Part 1 全 6 問 A（2026-07-17）。

---

## 論理削除の表現

| ID | ルール | 根拠 |
|---|---|---|
| **BR-U5-01（論理削除・物理削除しない）** | 出題停止は **`items.retired_at`**（**NULL=現役 / ISO8601=廃止時刻**）で表す。**行は削除しない**。過去の `pairs`/`judgments`/`likert_responses` は一切変更しない＝**それまでの判定結果は有効のまま**。<br>**物理削除が不可である構造的理由**: `pairs.item_left`/`item_right` は **`REFERENCES items(item_id)` の FK** を持つ（0001_init）。削除は FK 違反か、回避しても ExportBundle の自己完結性（BR-U3-07）破壊を招く。**論理削除が唯一の正解**。 | Q1=A / 要件 |
| **BR-U5-13（`retired_at` は現在状態・`admin_log` が履歴の正）** | `retired_at` は**現在状態**を表す単一の値。復活（BR-U5-07）は NULL に戻すため、**廃止→復活→再廃止の履歴はカラム単体には残らない**（最後の廃止時刻のみ）。**履歴の正は `admin_log`**（`item_retire`/`item_unretire` の時系列, BR-U5-11）。著作権対応の証跡としてはこの役割分担で足りる（追加実装は不要）。 | Q4=A 補足 |
| **BR-U5-12（Item 契約は不変・廃止を投入経路に持ち込まない）** | **`Item`（投入契約）に `retired_at` を持たせない**。廃止状態は D1 の列と専用 API のみが扱う。→ `pool_ingest` の経路からは**構造的に廃止・復活ができない**（BR-U5-05/08 を型で担保）。フィルタは SQL 側（`WHERE retired_at IS NULL`）で効かせ、Python の `Item` は現状のまま。 | Q3=A / Q4=A |

## 参照先の分離（★最重要・構造で固定する）

| ID | ルール | 根拠 |
|---|---|---|
| **BR-U5-02（読み取り経路を「関数の分割」で固定する）** | 本設計は「**active のみを読む経路**」と「**全件を読む経路**」の**二本立てで初めて成立**する。これを規約ではなく**関数の分割**で固定する:<br><br>　🔒 **`list_items()` は全件のまま・シグネチャも挙動も変更しない**<br>　🆕 **`list_active_items()` を新設**（`WHERE retired_at IS NULL`）。**新規セッション系と充足判定のみが呼ぶ**<br><br>**❌ 禁止（Code Gen の劣化経路）**: **既存の `list_items()` 自体に active フィルタを足すこと**（引数 `active_only=True` の既定値化を含む）。これをやると**要件の両輪が同時に壊れる**:<br>　(1) **エクスポートの `items` が縮む** → 自己完結性（judgments の item ⊆ items）が破れ **PU3-3 違反 → U4b 破壊**（＝「それまでの結果は有効」が壊れる）<br>　(2) **旧セッションのフォールバック導出が変わる** → **「新規のみ反映」が破れる**（＝進行中セッションが壊れる）<br>PU3-3 は検出網として残るが、**テストで検出しにくい仕様は明文で固定する**（MM 式・α 適用位置と同系）。 | Q2/Q5/Q6=A |
| **BR-U5-02a（呼び出し先の割当）** | **`list_active_items()` を呼ぶ（active のみ）**<br>　① 新規セッションのペア生成（`generate_pairs`）※**練習ペアも同一呼び出し由来**ゆえ自動的に効く（下記注）<br>　② 新規セッションの Likert ターゲット選定（`select_likert_targets`）<br>　③ 充足判定（ingest の `sufficiency_warnings` / issue のゲート）<br><br>**`list_items()` を呼ぶ（全件・現状維持）**<br>　④ `build_view` の `bodies` 写像（**進行中セッションの既存ペア列を解決するために必須**）<br>　⑤ 旧セッション（`likert_targets IS NULL`）の Likert 導出フォールバック（BR-U5-04）<br><br>**SQL 直参照（全件・現状維持）**<br>　⑥ エクスポートの `items`（`read_export_rows("items")`）＝**自己完結性 BR-U3-07 に必須**・body 非含有ゆえ著作権上安全<br>　⑦ U3 の winrate 集計（過去判定の事実を表す・変更しない） | Q2/Q5/Q6=A |
| **BR-U5-02b（練習ペアに別経路はない・調査済み）** | 練習ペアは**専用の素材経路を持たない**。`generate_pairs` が `practice_pairs + session_pairs` を**同一プール・同一呼び出し**で生成し、**先頭 `practice_pairs` 件を練習とする**だけ（BR-10）。→ **active フィルタは練習ペアにも自動的に効き、漏れはない**（`assignment.py` で確認, 2026-07-17）。 | 調査確認 |

## 新規セッションのみ反映

| ID | ルール | 根拠 |
|---|---|---|
| **BR-U5-03（進行中セッションは不変）** | 廃止の反映は**新規セッション開始時のみ**。ペア列はセッション開始時に一括確定・保存済み（U2 既存挙動）ゆえ、**進行中セッションは配信段の改修なしで不変**。<br>**受容されたトレードオフ（要件確認済み）**: 進行中の参加者には廃止 item の本文が**出続ける**。露出が止まるのは当該セッションが**完了する**か**非アクティブ閾値（既定 48h）で除外される**まで。即時停止が必要になった場合は配信段でのスキップが別途必要（**本 FD のスコープ外**）。 | Q2 前提 / 要件 |
| **BR-U5-04（Likert ターゲットの保存化・★核心）** | Likert ターゲットを**セッション開始時に確定し `sessions.likert_targets`（JSON 配列）へ保存**する。**ペア列と同じ「開始時確定」原則に揃える**（導出と保存の非対称という既存の設計の歪みを解消＝本要件がなくても正しい形）。<br>**単一アクセサに集約（必須）**: 現在 `select_likert_targets` の導出は **3 箇所に散在**（`build_view` / `check_complete` / `submit_likert`）。**全箇所を単一のアクセサ経由に統一**する。一部だけ保存値に切り替えると「**表示されたターゲットを送信すると拒否される**」不整合が生じる（表示=保存値・検証=導出値のずれ）。<br>**後方互換**: `likert_targets IS NULL`（U5 以前に開始した進行中セッション）は **`list_items()`（全件）から導出**にフォールバック＝**従来挙動を完全再現**し進行中を壊さない。 | Q2=A |

## 凍結ガードとの関係

| ID | ルール | 根拠 |
|---|---|---|
| **BR-U5-05（廃止は凍結ガード BR-U4a-03 の対象外）** | **`retired_at` の付与・解除は BR-U4a-03（参照済み item への UPDATE 拒否）の対象外**とし、**専用経路でのみ**許可する。<br>**根拠**: 凍結ガードの目的は BR-U4a-03 が明記するとおり「**判定後の本文・層の書換で過去判定の解釈が壊れる**（研究データ完全性）」の防止。`retired_at` は **`body`/`layer` を変えない**＝過去判定の解釈は一切壊れない。よってガードの趣旨に反しない。<br>**凍結は維持**: `body`/`layer`/`body_ref` の UPDATE は**引き続き拒否**（`insert_items` のガードは無改修）。廃止は別関数・別エンドポイント（BR-U5-06/07）。 | Q3=A |
| **BR-U5-08（再投入は復活させない）** | 廃止済み item を `pool_ingest` で再投入しても **`retired_at` は変更しない**（upsert は `layer`/`body`/`body_ref` のみ更新＝現行 `ON CONFLICT DO UPDATE` のまま）。復活は**専用経路のみ**（BR-U5-07）。投入と廃止の責務分離を保つ。 | Q3=A / Q4=A |

## 操作（API / CLI）

| ID | ルール | 根拠 |
|---|---|---|
| **BR-U5-06（廃止操作・冪等）** | **`POST /admin/items/retire`**（既存 Basic 認証の背後）。入力 = `item_id` 列。**冪等**: 既に廃止済みの item は **no-op**（`retired_at` を上書きせず**初回の廃止時刻を保持**＝経緯が後から書き換わらない）。存在しない `item_id` は `not_found` として報告（エラーにせず部分成功を返す）。結果 = `{ok, retired: int, already_retired: [...], not_found: [...]}`。 | Q4=A |
| **BR-U5-07（復活操作）** | **`POST /admin/items/unretire`**（対称な別ルート）。`retired_at=NULL` に戻す。誤操作の回復用。冪等（現役の item は no-op）。CLI は `scripts/pool_retire.py --unretire`。 | Q4=A |
| **BR-U5-11（監査ログ）** | `admin_log` に **`item_retire` / `item_unretire`** を記録（対象 `item_id` 列挙・件数・結果）。**これが廃止履歴の正**（BR-U5-13）。著作権対応の証跡として **API 経由を正とする**（D1 直操作は証跡が残らず不可）。**本文（`body`）はログに出さない**（既存の秘匿方針を維持）。 | Q4=A |

## 充足判定・エクスポート

| ID | ルール | 根拠 |
|---|---|---|
| **BR-U5-09（充足判定は active 母数）** | プール充足判定（BR-U4a-05 の三点セット: ① 総数 ≥ ceil(2×session_pairs/k) ② 4 層非空 ③ 層間供給）の**母数を active のみ**とする（`list_active_items()`）。**出題できない作品を母数に入れない**（入れると「発行はできるが割当が偏る/失敗する」状態を作る）。<br>ingest の `sufficiency_warnings`（マージ後の見込みプール, BR-U4a-05）も **active ∪ 入力**で評価。<br>**廃止の結果ゲートを割ったら発行拒否が正しい挙動**（BR-U4a-12）＝運用者に補充を促す。 | Q5=A |
| **BR-U5-10（エクスポートは全件・`retired_at` 非出力）** | エクスポートの `items` は**廃止分も含む全件**（**自己完結性 BR-U3-07 = judgments の item ⊆ items に必須**）。`ExportItem` は **`body` 非含有**ゆえ廃止分を出しても著作権上安全。<br>**`retired_at` はエクスポートに含めない**: BT 集計は使わず、含めると `ExportItem` の形式変更＝**BR-U3-07 により `EXPORT_FORMAT_VERSION` の版上げ必須**となり **U4b に波及**（`extra="forbid"` + 版検証が既定エラー）。**`EXPORT_FORMAT_VERSION` は 1.0.0 据え置き・U3/U4b は無変更**。廃止の事実と時刻は D1・`admin_log`・運用記録に残る。将来 retired の分析が必要になった時点で版上げする。 | Q6=A |

## 検証・エラー処理サマリ

| 状況 | 挙動 |
|---|---|
| 参照済み item の廃止 | **許可**（BR-U5-05: 凍結ガード対象外。body/layer は不変ゆえ過去判定の解釈は壊れない） |
| 参照済み item の body/layer 更新試行 | **引き続き拒否**・投入全体を中断（BR-U4a-03 は無改修） |
| 既に廃止済みの item を廃止 | **no-op**・`already_retired` に列挙（初回時刻を保持, BR-U5-06） |
| 存在しない item_id を廃止 | `not_found` に列挙（エラーにせず部分成功, BR-U5-06） |
| 廃止済み item を pool_ingest で再投入 | `layer`/`body`/`body_ref` は更新されるが **`retired_at` は不変**＝廃止のまま（BR-U5-08） |
| 廃止で充足を割った状態で token_issue | **発行拒否**（BR-U5-09 / BR-U4a-12）。`gate_errors` に不足内訳 |
| 進行中セッションが廃止 item を含む | **そのまま出題される**（BR-U5-03・要件で受容済み）。`bodies` は全件参照ゆえ画面は壊れない |
| U5 以前に開始した進行中セッション（`likert_targets IS NULL`） | **全 items から導出にフォールバック**＝従来挙動を完全再現（BR-U5-04） |
| 廃止 item が過去の judgments を持つ | **BT 集計にそのまま含まれる**（U4b 無変更・「それまでの結果は有効」） |
| 練習ペアへの反映 | 同一プール・同一 `generate_pairs` 由来ゆえ**自動的に active のみ**（BR-U5-02b） |
