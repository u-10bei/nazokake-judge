# U1 Business Rules — 共有基盤

割当・状態・データ整合に関する業務ルール、検証、エラー処理。パラメータの暫定値は開発・テスト用（Negotiable、実験計画で調整）。

---

## 割当・セッション構成ルール

| ID | ルール | 根拠 |
|---|---|---|
| **BR-01** | 1 ペアは相異なる 2 作品（`a ≠ b`）。セッション内で**同一ペアの重複を禁止**。 | 追加規則1 |
| **BR-02** | **同一項目のセッション内出現回数は高々 k 回**（`max_item_occurrence_k`、暫定 **k=3**）。同一作品の反復出現が記憶・慣れで判定を汚染するため。 | 追加規則1 |
| **BR-03** | **層間ペア比率 ≥ `cross_layer_min_ratio`**（暫定下限 **0.65**）。層は割当で区別しつつ単一パラメータに留める。 | Q2=A, FR-03 |
| **BR-04** | 露出カウント導出時、**`last_active_at` が閾値を超えて古い in_progress セッションを除外**（`inactive_threshold`、暫定 **48 時間**）。離脱データの空カウント滞留を防ぐ。 | Q5=A 追加規則 |
| **BR-05** | **構成不能条件（プール過小で本番ペアを規定数構成できない 等）は、参加者アクセス時ではなく `pool_ingest`／設定検証時の事前チェックで排除**する。参加者に見せてよいのはベストエフォートまで。 | Q6=A 追加規則 |
| **BR-06** | 露出目標を満たせない軽微な不均衡は**ベストエフォート + ログ警告**で許容（データは汚さない）。 | Q6=A |
| **BR-07** | 各ペアの **A/B 提示位置（`item_left`/`item_right`）を一様ランダムに割当・記録**（カウンターバランス）。`choice` は left/right の選択を表す。 | 追加規則2 |

## データ整合・状態ルール

| ID | ルール | 根拠 |
|---|---|---|
| **BR-08** | ペア判定は (`token`,`pair_id`) で**冪等**（重複送信は 1 件）。練習試行は集計対象外。 | US-P03, H-3 |
| **BR-09** | トークン状態遷移は `unused → in_progress → completed` の一方向。completed への再アクセスは新規回答不可。 | US-P01/P07 |
| **BR-10** | **練習/本番の判定はサーバが PairSequence の構成（`is_practice`/index）から決定**。クライアント申告を信用しない。 | H-3 |
| **BR-11** | Item は**層ラベル必須**。欠落は投入拒否（US-R06）。 | XC-01 前提 |
| **BR-12** | 全永続化はパラメータ化クエリ（SQLi 対策）。 | XC-03 |

## 検証・エラー処理

| 状況 | 挙動 |
|---|---|
| 無効/存在しないトークン | エラー表示、判定画面へ進めない（US-P01） |
| completed トークンの再アクセス | 「完了済み」表示、新規回答拒否 |
| 層ラベル欠落の投入 | `pool_ingest` でエラー・拒否（BR-11） |
| プール過小（構成不能） | `pool_ingest`／設定検証で事前エラー（BR-05） |
| 無回答での送信 | 拒否（US-P03） |
| 重複判定送信 | 冪等に 1 件（BR-08） |

## パラメータ暫定値（開発・テスト用、Negotiable）

| パラメータ | 暫定値 | 決定タイミング |
|---|---|---|
| `session_pairs`（本番ペア） | 40 | 実験計画確定時 |
| `practice_pairs` | 3 | 同上 |
| ブリッジ Likert 項目数 | 10 | プール確定後 |
| `cross_layer_min_ratio` | 0.65 | 実験計画 |
| `max_item_occurrence_k` | 3 | 実験計画 |
| `inactive_threshold` | 48h | 運用時 |
| `α`（露出偏り許容係数） | **0.7**（較正確定 2026-07-13） | Code Generation 較正シミュレーション |
| `S`（露出偏り評価の累積セッション数） | **30**（較正確定 2026-07-13） | 同上 |
| 重み関数指数 `p`（`weight = 1/(有効露出+1)^p`） | **3**（較正確定 2026-07-13） | 同上 |

**露出偏り許容範囲の述語（BR-03 の露出側・XC-01 チェックリスト対応）**: 適格項目（非アクティブ除外後）の露出回数について `max − min ≤ max(2, α × mean)` を、**S セッション累積シミュレーション後**の統計的不変条件として満たす（述語の形は確定）。詳細は `business-logic-model.md` §5「P-1 露出偏り許容範囲の述語定義」。

**較正確定（2026-07-13）**: 本番規模プール（95 件 = pro30/ai20/edit30/rule15、本番 40 ペア/セッション、k=3、層間 0.65）で S セッション累積・20 試行の最悪 gap を評価。暫定 α=0.5（p=1）は S=30 で不成立（必要 0.713）だったため、**重み指数を p=3 に強化**（`weight = 1/(有効露出+1)³`）。p=3 で S=30 最悪 α≈0.475 となり、約 1.5 倍のマージンで **α=0.7 / S=30** を確定。比較グラフへの副作用なし（p=1〜4 で平均ユニーク相手数 ~22・ユニーク辺数 ~1045・**連結成分は常に 1**）を確認済み。実装 = `backend/domain/assignment.py`、検証 = `tests/pbt/`（P-1 は本番規模プールで評価）。
