# U3 NFR Requirements — 研究者・管理（admin）

U1〜U2 の NFR・U4a-NFR-01〜12 を前提に、U3 固有の非機能要件を **U3-NFR-NN** で定義する。U3 は**既存 Basic 認証境界の再利用・読み取り専用**のため差分は小さく、最大論点は**エクスポート（トークン付き研究データ）の秘匿**。
**技術決定の詳細**: `tech-stack-decisions.md`（TSD-U3-01〜05）を参照。

---

## 1. セキュリティ（最重要 / Security 拡張は No・通常実装で必須）

- **U3-NFR-01（認証一本化・再利用）**: 管理 UI（`GET /admin/`）・管理 API（`/admin/progress`・`/admin/winrates`・`/admin/export`）は**すべて U4a の Basic 認証チョークポイント（単一資格 `ADMIN_BASIC_*`）の背後**（BR-U3-01 / DP-U4a-01）。**新規資格・新規認証を作らない**。HTTPS 強制。定数時間比較（既存 AuthGuard 流用）。（Q2）
- **U3-NFR-02（管理 HTML の assets 非配置）**: **管理 HTML/JS を Static Assets（`frontend/`）に置かない**。assets は公開配信のため認証境界外へ漏れる。**`src/backend/admin/` 埋め込み文字列として Worker が Basic 認証背後で返す**（BR-U3-02）。（Q2）
- **U3-NFR-03（エクスポート秘匿）**: エクスポートは**トークン付き回答データを Basic 認証背後で返す**（評価者相対性分析に実トークンが必要, US-R02）。(i) **応答に `Cache-Control: no-store`**（キャッシュ滞留防止）。(ii) **ログにトークン・本文を出さない**（AdminLog 秘匿, BR-U3-08）。(iii) **body はエクスポートに含めない**（未公表刺激, BR-U3-07 / NFR-08）。(iv) エクスポートファイルの保管（研究者ローカル）は運用責任＝リポジトリ管理外（NFR-08）。**トークンの仮名化はしない**（分析・公表側 U4b の選択に委ねる）。（Q1）
- **U3-NFR-04（CORS なし）**: 管理 UI/API は**同一オリジン**（ブラウザから同一サブドメインの `/admin/*`）ゆえ **CORS を設けない**。U2 NFR で予告した「U3 での CORS 拡張検討」は**不要と決着**。（Q2）
- **U3-NFR-05（基本衛生）**: 全集計 D1 アクセスはパラメータ化クエリ（SQLi 対策, BR-12/XC-03）。U3 は読み取り専用だが読み取りクエリも文字列連結を禁止。（XC-03）

## 2. 性能（最小限）

- **U3-NFR-06（SLO 非設定）**: 集計クエリ（進捗・勝率・エクスポート join）に**明示的な数値 SLO を設定しない**（U1/U2 と同方針）。参考目安「進捗/勝率 < 1s・エクスポート < 数秒」。想定規模（判定 ≈ 2,000 行 + items 95）で単一集計クエリは実質瞬時。**インデックス追加・マテリアライズ・ページングは不要**。（Q3）

## 3. 信頼性（軽微・読み取り専用）

- **U3-NFR-07（読み取り専用・整合は集計の正しさで担保）**: U3 は**参加者データを書き込まない**（冪等・原子性・トランザクションは対象外）。データ整合の要件は「**集計の正しさ**」＝練習除外の出力段保証（BR-U3-03）・winrate 定義整合（BR-U3-05）・ExportBundle 自己完結（BR-U3-07）に集約し、Testability（§5）で検証する。（FD 由来）

## 4. 可観測性（最小限・流用）

- **U3-NFR-08（秘匿ログ）**: AdminLog（U4a）を再利用し、管理操作（進捗参照・エクスポート実行）を構造化出力。**トークン生値・本文は非出力**。監視基盤なし（stdout, U1-NFR-10 と同方針）。（BR-U3-08）

## 5. テスト容易性

- **U3-NFR-09（単体/統合の振り分け）**: **純粋な整形ロジックは example ベース単体テスト**（CSV 直列化・ProgressView/WinrateRow 変換・ExportBundle 組立）、**D1 集計・認証は統合テスト**（`tests/integration/` 流用・`/admin/*` 越し）: PU3-1（練習除外の出力段保証）・PU3-2（winrate 定義整合）・PU3-4（進捗カウント整合）・PU3-5（認証 401）。（Q4）
- **U3-NFR-10（PBT 適用範囲）**: PBT 強制セット（PBT-02/03/07/08/09）は U3 では**該当が薄い**（新規の純粋不変条件が乏しく、ラウンドトリップ対象なし）。**PU3-3（ExportBundle 自己完結: judgments に現れる item_id ⊆ items）のみ PBT 候補**（生成データで包含関係を反例探索）。**非該当を明記**（Requirements Q14 Partial の監査可能性）。（Q4）

## 6. 可用性（Usability・限定）

- **U3-NFR-11（管理 UI 水準）**: **デスクトップ主・日本語**（研究者 P-RSCH は PC 利用）。**非 BT の明示を可用性要件として固定**（暫定勝率が正式 BT 推定と誤読される事故＝反復判定装置運用の現実的な認知リスクを防ぐ, BR-U3-05）。アクセシビリティは合理的水準（セマンティック HTML・キーボード操作可）、正式 WCAG 準拠は非目標。モバイルは破綻しない程度（XC-04 からの意識的逸脱を FD で記録済み, BR-U3-10）。（Q5）

---

## 7. 非目標（意図的な非要件）

- **Data/Migration**: **N/A — U3 は読み取り専用**（新規テーブル・DDL 変更・migration なし）。
- **Scalability / Resiliency**: N/A（単独研究者・小規模）。
- **数値 SLO・クエリ最適化**: 設定しない（U3-NFR-06）。
- **トークン仮名化**: しない（U3-NFR-03、実トークンを認証背後で返す。仮名化は分析側の選択）。
- **CORS**: 設けない（U3-NFR-04、同一オリジン）。
- **正式 WCAG 準拠・モバイルファースト**: 目標としない（U3-NFR-11、研究者 PC 利用）。
- これらを意図的非要件として明示記録（後の再燃防止。U1/U2/U4a と同流儀）。
