# U4b Business Rules — BT 集計スクリプト（bt_aggregate）

U1〜U3・U4a を前提に、U4b 固有の規則を **BR-U4b-NN** で番号付けする。U4b は**オフライン pure-Python**（Worker 非依存）で、U3 の ExportBundle を入力とする統計推定。

---

## BT 推定

| ID | ルール | 根拠 |
|---|---|---|
| **BR-U4b-01（MM 推定・擬似データ正則化）** | BT 強度は **MM アルゴリズム（Hunter 2004 の標準反復）を純 Python** で最尤推定する（依存ゼロ）。正則化は**擬似データ拡張**で実装する（BR-U4b-03）: 各観測ペア (i,j) に仮想引き分け α 件を加える → `w̃_ij = w_ij + α/2`、`ñ_ij = n_ij + α`。これにより `w̃_i = w_i + (α/2)·d_i`（`d_i` = 観測対戦相手数）、`ñ_ij` は観測ペアのみで加算。**更新式は素の Hunter**: `π_i ← w̃_i / Σ_j ñ_ij/(π_i+π_j)`（分母は観測ペアのみ和）。`θ_i = log π_i` を出力。反復回数・収束閾値はパラメータ化。**決定論**（初期値・反復固定 → 同一入力同一出力）。<br>**注記**: 分子に一律 α を足すだけ（`(w_i+α)/…`）の形は**いかなる拡張データの最尤にも対応しない別の regularizer**であり不採用。擬似データ版は疎な新作ほど相対的に強く縮む（狙いの性質）+ MM の単調収束が擬似データに対して自明に保たれる。単調性（PU4b-1）は両式で通るためテストで検出されない＝定式化を仕様で固定する。 | Q1 / Q2 / P-6 |
| **BR-U4b-02（識別可能性・最大連結成分）** | 観測ペアを辺とする無向グラフの**連結成分を検出**し、**非連結なら warnings に記録 + 最大連結成分のみを推定対象**とする。除外された item は結果から**黙って消さない**（BR-U4b-08）。コンポーネント間スコアは比較不能である旨を明記。 | Q2 |
| **BR-U4b-03（観測ペア限定スムージング）** | 正則化（各観測ペアに**仮想引き分け α 件**＝各側に α/2 勝ちを配分, BR-U4b-01）は **観測されたペアのみに適用**し、**全 item ペアには張らない**。全ペアに張ると比較グラフが人工的に完全連結化し連結成分検出（BR-U4b-02）が無意味化（非連結という研究上重要な事実を隠蔽）するため。観測ペア限定で「発散防止・完全勝敗の有限化」と「連結性診断」を両立。 | Q2 明確化 |
| **BR-U4b-04（正規化）** | 位置の任意性を除くため **推定対象（最大連結成分）内で `Σθ = 0`** に正規化する（全 item 集合ではなく成分内）。 | Q2 / Q7 |

## Likert 較正

| ID | ルール | 根拠 |
|---|---|---|
| **BR-U4b-05（較正アンカー・target_ref=item_id）** | ブリッジ Likert を**較正アンカー**として BT 尺度を解釈可能尺度へ写像する（推定には組み込まない）。**`likert.target_ref` = 評定対象の `item_id`**（U2 実装で確認: `select_likert_targets` は item_id 列を返し `submit_likert` が `target_ref ∈ targets` を検証）。この対応で結合。アンカー = {推定対象 item ∩ Likert 平均あり ∩ `target_ref ∈ items`}。`target_ref ∉ items` のアンカーは**較正から除外 + 警告**（自己完結保証は judgments のみで likert には及ばない）。(平均 Likert, θ) を**単回帰（閉形式）**で当て `calibrated=(θ−a)/b`。 | Q3 / US-R04 |
| **BR-U4b-06（較正スキップ条件）** | **アンカー 2 件未満 / アンカーの θ 分散ゼロ（全同点）/ slope≈0** のいずれかなら**較正をスキップ**し、生 θ のみ + 警告。誤った較正で解釈を歪めない。 | Q3 |

## 出力（BTResult）

| ID | ルール | 根拠 |
|---|---|---|
| **BR-U4b-07（除外 item の可視化）** | 非連結で推定から除外された item も **`items` 配列に残し `bt_score=null` + `component`（成分 ID）** で除外を可視化する（黙って消えると「新作の位置確認」で見落とす）。 | Q4(1) |
| **BR-U4b-08（U3 突合可能性）** | `matches`/`wins` の定義は **U3 winrate と同一**（matches=出場数、wins=勝ち数、`choice=A`→item_left 勝ち）。同一エクスポートで U3 ダッシュボード（PU3-2）と突合できる。 | Q4(2) / BR-U3-05 |
| **BR-U4b-09（source エコーバック）** | BTResult に **`source: {schema_version, exported_at}` をエコーバック**する。反復判定装置（複数スナップショットで回す）で、BT 結果がどの時点のエクスポート由来か**結果ファイル単体で追える**（取り違え防止）。 | 横断指摘 |
| **BR-U4b-10（層の付与・相対位置）** | 各 item に `layer`（items 由来）を付与し、**新作のプロ水準に対する相対位置**を層別サマリ（層ごとのスコア分布）で示す。 | US-R04 |

## 入力・版管理

| ID | ルール | 根拠 |
|---|---|---|
| **BR-U4b-11（版検証・既定エラー）** | 入力 `schema_version` と `EXPORT_FORMAT_VERSION` の**不一致は既定でエラー（終了）**。BR-U3-07 が版上げを伴う契約変更と定義済みゆえ、不一致を警告で通すと契約の意味が消える。**`--allow-version-mismatch`** で明示的に緩和可（その場合 `warnings` に記録）。 | Q5 |
| **BR-U4b-12（本番のみ・再フィルタなし）** | judgments は **U3 が本番のみに出力段で保証済み**（BR-U3-03）。U4b は**再フィルタしない**（U3 保証を信頼）。練習除外の二重責務を持たない。 | Q5 / BR-U3-03 |
| **BR-U4b-13（scripts 分離・依存最小）** | `scripts/` 配下・**アプリ本体（Worker）と分離**・**追加依存なし（標準ライブラリのみ）**。`schema/`（ExportBundle 型・EXPORT_FORMAT_VERSION）のみ import。 | Q6 / US-R04 |

---

## 検証・エラー処理サマリ

| 状況 | 挙動 |
|---|---|
| schema_version 不一致 | 既定エラー終了（--allow-version-mismatch で warnings 続行, BR-U4b-11） |
| 比較グラフ非連結 | warnings + 最大連結成分のみ推定・除外 item は bt_score=null+component（BR-U4b-02/07） |
| 完全勝ち/完全負けの item | 観測ペア限定正則化で有限スコア（BR-U4b-03） |
| アンカー不足/θ 分散ゼロ | 較正スキップ + 生 θ + 警告（BR-U4b-06） |
| target_ref ∉ items | 該当アンカーを較正から除外 + 警告（BR-U4b-05） |
| 空 judgments | 推定不能 → warnings + 空/生データ返却（実装で明示） |
