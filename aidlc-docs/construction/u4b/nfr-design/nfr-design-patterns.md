# U4b NFR Design Patterns — BT 集計スクリプト（bt_aggregate）

U4b-NFR-01〜13 を設計パターン **DP-U4b-NN** に落とす。U4b は**オフライン pure-Python・純関数中心**ゆえ、DP は「正しさと再現性を構造で守る」に集約。案 A′（pure-Python・追加依存なし, BR-U4b-13）。

---

## DP-U4b-01: 正準集計の 3 点セット（決定論 + 集計の正しさ・最重要）
行順序不問決定論（U4b-NFR-01）は**ソート順**と**集計の正しさ**の両問題。**正準集計を 1 箇所の純関数に集約**し、規約でなく**通過必須の関数**にする（Code Gen での劣化経路＝呼び出し側の生 dict 集計を塞ぐ）:
1. **ペアキー正準化**: 無順序ペア `(i,j)` を **`sorted((i,j))` の tuple** に正規化してから `n_ij` を数える。**勝敗は item 単位**（`w_i` = その item が勝った回数）で向き非依存に集計。→ `n_ij` 分裂・**擬似データ α の二重配分**を排除。
2. **item 列の正準ソート**: 推定対象 item を `item_id` 昇順で固定（初期値・反復・Σ 対象順を固定）。
3. **Σ 加算順の固定**: MM 更新の分母 Σ・θ 正規化の Σ を**ソート済み順**で加算（浮動小数の加算順依存を除去）。
- **なぜ集計の正しさか**: `(i,j)` と `(j,i)` を別キーで数えると n_ij が分裂し、両向きに α が乗って**疎ペアほど正則化が倍効き**＝**新作（疎）のスコアが系統的に歪む**（装置の目的に直撃）。
- **検証**: **PU4b-2 の置換不変性 PBT のジェネレータに「judgments シャッフル + 左右反転（item_left/item_right 交換時 choice も A↔B 反転）」を含め**、3 点まとめて反例探索。
- 対応: U4b-NFR-01, BR-U4b-01/03, Q1。

## DP-U4b-02: 純関数分離 + 成分制限の切り出し
統計ロジックを**副作用なしの純関数**に分け、CLI は薄い I/O 境界（引数・ファイル・版検証・終了コード）に限る:
- `aggregate`（正準集計・DP-U4b-01）／`connected_components`／**`restrict_to_component`（最大連結成分への制限を独立純関数に切り出す）**／`fit_bt`（**成分非依存**＝渡された連結な集計を推定するだけ）／`calibrate`／`assemble_result`。
- **なぜ切り出すか**: 成分制限を CLI インラインにすると **PU4b-4（非連結→最大成分のみ）が CLI 越しでしか書けない**。独立関数にすれば **PU4b-4 は純関数合成（`connected_components → restrict_to_component → fit_bt`）で直接検証**でき、`fit_bt` の PBT（PU4b-1/2/3）は**連結入力前提で単純化**される。
- すべて `schema/`（ExportBundle/BTResult 型）のみ依存。Worker/D1 非依存・層の逆流禁止。
- 対応: U4b-NFR-06/07, BR-U4b-02, Q2。

## DP-U4b-03: 終了コード契約（失敗と warnings の分離）
- **純関数はデータ or 例外を返し、終了コード決定は CLI に集約**する。これで終了コード契約（U4b-NFR-11）のテストが **CLI unit 1 箇所に閉じる**。
- **失敗（非0）**: 入力ファイル不在・JSON パース不能・ExportBundle 検証失敗・版不一致（既定）。
- **成功（0）**: 正常 + warnings 系（非連結・較正スキップ・未収束・除外 item・版不一致緩和）。
- **warnings 二重表示**: `BTResult.warnings`（機械可読・CI）+ 人間可読テーブル冒頭（日本語・対話運用）。例外は CLI 境界で捕捉して終了コードへ写す。
- 対応: U4b-NFR-03/10/11, BR-U4b-11, Q3。

## DP-U4b-04: 決定論・数値安定（既存 NFR の設計化）
- 乱数不使用・固定初期値 π=1・パラメータ（α/tol/max_iter）既定固定 + 使用値を BTResult に記録（監査）。
- 数値安定: 観測ペア限定正則化 α で分母>0 保証（ゼロ割回避）・θ 正規化は**最大連結成分内 Σθ=0**。
- 未収束は `converged=false` + warnings で結果出力（exit 0）。
- 対応: U4b-NFR-02/03/04/05, BR-U4b-03/04。

## 導入しない設計部品（意図的な非採用・U1〜U3 と同方針）
| 部品 | 非採用理由 |
|---|---|
| キャッシュ | 単発オフライン・純計算。 |
| キュー / 並列 | 小規模・逐次で数秒未満。 |
| サーキットブレーカ / リトライ | 外部依存なし（ファイル入出力のみ）。 |
| 分散ロック | 単発実行・書き込み共有なし。 |
| numpy / scipy | pure-Python 標準ライブラリで十分（BR-U4b-13）。 |
