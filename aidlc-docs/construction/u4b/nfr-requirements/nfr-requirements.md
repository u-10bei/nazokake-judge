# U4b NFR Requirements — BT 集計スクリプト（bt_aggregate）

U1〜U3・U4a の NFR を前提に、U4b 固有の非機能要件を **U4b-NFR-NN** で定義する。U4b は**オフライン pure-Python**（Worker/D1 非依存）ゆえ、固有論点は **統計推定の再現性・数値安定性**と **PBT 中心のテスト戦略**。
**技術決定の詳細**: `tech-stack-decisions.md`（TSD-U4b-01〜06）を参照。

---

## 1. 再現性・数値安定性（最重要 / Reliability 派生）

- **U4b-NFR-01（行順序不問の決定論）**: 「**同一データ（行順序不問）→ 同一 BTResult**」を保証する。推定前に **items・観測ペア・judgments 集計を `item_id` で正準ソート**してから反復する（純 Python の浮動小数演算は加算順に依存＝D1 エクスポート行順が変わると θ が最下位桁で揺れうるため）。反復判定装置＝複数スナップショット突合の運用で、行順序という非本質的差でスコアが揺れないことが診断上の要件。**PU4b-2 の PBT に置換不変性**（judgments シャッフルで BTResult 一致）を含める。（Q1）
- **U4b-NFR-02（完全決定論・乱数不使用）**: 固定初期値 π=1・固定反復順・パラメータ（α・収束閾値・最大反復）既定固定。乱数不使用（P-6/BR-U4b-01）。同一入力・同一パラメータ → 同一出力。（Q1）
- **U4b-NFR-03（収束の明示・未収束の扱い）**: `converged`/`iterations` を BTResult に含める。**未収束（最大反復到達）は `converged=false` + warnings で結果を出力し終了コード 0**（エラーにしない＝θ は使用可能な近似・収束フラグで研究者が判断）。（Q1）
- **U4b-NFR-04（数値安定性）**: π は正値維持・log/除算のゼロ割回避（観測ペア限定正則化 α で分母>0 保証, BR-U4b-03）・θ 正規化は**最大連結成分内**（BR-U4b-04）。（Q1）
- **U4b-NFR-05（パラメータの監査可能性）**: α・収束閾値・最大反復は CLI で上書き可・**既定値と使用値を BTResult に記録**（監査再現性）。（Q1）

## 2. テスト容易性（PBT 中心）

- **U4b-NFR-06（PBT/example 振り分け）**: **BT 推定の不変条件は PBT（Hypothesis, PBT-03 系）**:
  - PU4b-1 単調性（A 常勝→θ_A>θ_B、**擬似データ正則化 ON でも保存**）／PU4b-2 決定論**+置換不変性**（judgments シャッフルで一致）／PU4b-3 正規化（**最大連結成分内 Σθ=0**）／PU4b-4 識別可能性（非連結→警告+最大成分・除外 item 可視化）／PU4b-5 較正係数復元（既知線形データ・example+PBT）。
  - **単体（example）**: CLI 入出力・版検証（PU4b-7）・**U3 突合（PU4b-6・同一エクスポートで matches/wins 一致）**・出力形式。（Q2）
- **U4b-NFR-07（ジェネレータ設計）**: PBT ジェネレータは **連結/非連結グラフを両方生成**する（「非連結を意図的に含む」が PU4b-4 の反例探索の要）。プール・比較列・Likert アンカーを生成。（Q2）
- **U4b-NFR-08（PBT 適用範囲）**: 強制セット中 U4b の主対象は **PBT-03（不変条件）**。**PBT-02（ラウンドトリップ）は非該当**（U4b は入力→出力の一方向変換で逆写像を持たない）。非該当を明記（Requirements Q14 Partial の監査可能性）。（Q2）

## 3. 性能（最小限）

- **U4b-NFR-09（SLO 非設定）**: 明示的な数値 SLO を設定しない（U1〜U3 と同方針）。参考目安「実行 < 数秒」。MM は本規模（items 約 95・判定数千・観測ペア限定）で純 Python でも数秒未満収束。**反復上限で無限ループを防ぐ**。numpy 化・並列は不要（BR-U4b-13）。（Q3）

## 4. 可用性（CLI・限定）

- **U4b-NFR-10（CLI・出力）**: 引数=エクスポート JSON パス、オプション `--out`/`--alpha`/`--max-iter`/`--tol`/`--allow-version-mismatch`。**JSON（機械可読）+ 人間可読テーブル（層別・スコア降順）** を出力。**warnings を目立つ形で表示**（非連結・較正スキップ・版不一致緩和・除外 item・未収束）。日本語で要点。（Q4）
- **U4b-NFR-11（終了コード規約）**: **非0（失敗）= 版不一致（既定）・入力ファイル不在・JSON パース不能・ExportBundle 検証失敗**（CI 自動実行で失敗を exit code 検知できることが必要）。**0（成功）= 正常 + warnings 系（非連結・較正スキップ・未収束）**。（Q4）

## 5. セキュリティ（オフライン・最小衛生）

- **U4b-NFR-12（token 非参照・非含有）**: U4b は**オフライン**（ネットワーク不要）。**BT 推定も Likert 較正も token を一切使わない**（推定=item_left/item_right/choice、較正=target_ref/rating のみ）＝**U4b は token を読み飛ばし、出力にも含めない**。したがって **BTResult は token 非含有かつ U4b の計算経路自体が token 非参照**（再識別リスクを構造的に排除）。SQLi/認証等は非該当（D1・API なし）。（Q5）
- **U4b-NFR-13（ローカル保管の運用責任）**: 入力エクスポート・出力 BTResult のローカル保管は運用責任（NFR-08・リポジトリ管理外）。U4b は新規の公開面を作らない。（Q5）

## 6. 非目標（意図的な非要件）

- **Data/Migration**: N/A（D1 非依存・ファイル入出力のみ・DDL 変更なし）。
- **Availability / Scalability / Resiliency**: N/A（単発オフライン実行）。
- **数値 SLO・numpy 最適化・並列**: 設定しない（U4b-NFR-09）。
- **token 別内訳の出力**: しない（U4b-NFR-12、集計値のみ）。
- **PBT-02（ラウンドトリップ）**: 非該当（U4b-NFR-08、一方向変換）。
- これらを意図的非要件として明示記録（後の再燃防止。U1〜U3 と同流儀）。
